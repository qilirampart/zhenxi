> 版本：Build-2（2026-03-26）

## 一、环境要求

| 项目 | 要求 |
|------|------|
| 操作系统 | Windows 10 / 11 |
| Python | 3.9 或以上 |
| 依赖包 | 见下方安装说明 |

**首次使用前安装依赖（在项目目录下执行）：**

```cmd
cd E:\点众\自动化工具
pip install -r requirements.txt
```

> 如果网络不稳定，可加镜像源：
> ```cmd
> pip install -r requirements.txt -i https://pypi.tuna.tsinghua.edu.cn/simple
> ```

## 二、启动程序

打开命令提示符（CMD）或 PowerShell，执行：

```cmd
cd E:\点众\自动化工具
python main.py
```

启动后弹出主窗口，日志输出在终端中可见（可用于排查问题）。

**常见启动失败原因：**

| 错误提示 | 解决方法 |
|----------|----------|
| `ModuleNotFoundError: No module named 'PySide6'` | 执行 `pip install PySide6` |
| `ModuleNotFoundError: No module named 'cv2'` | 执行 `pip install opencv-python` |
| 窗口闪退 | 在终端查看报错信息，确认依赖齐全 |

## 四、静态模式使用流程

> 适用场景：提示词固定显示在画面中（如片头字卡、固定水印文字）

### 步骤 1：导入视频

1. 点击工具栏 **「导入视频」**
2. 选择本地视频（支持 `.mp4` `.mov` `.avi` `.mkv` `.flv` `.wmv` `.m4v`）
3. 导入成功后，左侧显示预览画面和视频信息

### 步骤 2：选择 OCR 引擎

在模式控制面板的 **「OCR 引擎」** 下拉框中选择：

| 选项 | 说明 |
|------|------|
| 视觉 API（云端） | 调用云端大模型视觉接口，识别质量高，需要网络和 API Key |
| 本地 PaddleOCR | 使用本地离线模型，无需网络，速度快（需提前下载模型）|
| 自动（本地优先） | 本地模型可用时走本地，否则自动切换 API（默认）|

> **推荐测试时选「视觉 API（云端）」**，确保 `runtime/api_config.json` 已配置好 API Key。

### 步骤 3：生成候选帧

**方式 A：自动生成（推荐）**
1. 点击 **「自动生成候选帧」**
2. 程序按清晰度 / 文字密度 / 稳定度评分，最多选出 5 帧
3. 候选帧显示在列表中，默认全部勾选

**方式 B：手动添加**
1. 拖动进度条定位到含提示词的画面
2. 点击 **「添加当前帧」**
3. 可多次添加，每帧带时间戳

**混合使用**：自动生成后手动补充遗漏帧，两种方式可叠加。

### 步骤 4：管理候选帧列表

- 点击列表条目左侧勾选框：控制该帧是否参与 OCR
- 点击条目：在预览区跳转到对应帧
- 点击 **「删除选中帧」**：移除当前选中帧

### 步骤 5：开始提取

1. 确认已勾选至少一帧
2. 按需勾选 **「保留截图」**（提取时同步保存帧图片）
3. 点击 **「开始提取」**
4. 右侧结果面板实时显示进度，提取完成后显示识别文字

### 步骤 6：查看截图

- 提取完成后，工具栏 **「打开截图目录」** 按钮激活
- 点击后自动打开截图保存目录（Windows 资源管理器）
- 截图默认保存在 `output/screenshots/` 下按视频文件名分组

## 五、API 配置

编辑 `runtime/api_config.json`：

```json
{
  "enabled": true,
  "api_key": "your-api-key-here",
  "base_url": "https://api.openai.com/v1",
  "model": "gpt-4o",
  "timeout": 30
}
```

**其他兼容的 API 服务（替换 `base_url` 和 `model`）：**

| 服务 | base_url | model 示例 |
|------|----------|------------|
| OpenAI | `https://api.openai.com/v1` | `gpt-4o` |
| Claude (Anthropic) | `https://api.anthropic.com/v1` | `claude-opus-4-6` |
| Gemini | `https://generativelanguage.googleapis.com/v1beta/openai` | `gemini-2.0-flash` |
| 豆包 | `https://ark.cn-beijing.volces.com/api/v3` | `doubao-pro-vision-32k` |
| DeepSeek | `https://api.deepseek.com/v1` | `deepseek-chat` |
| 智谱 GLM | `https://open.bigmodel.cn/api/paas/v4` | `glm-4v` |

## 六、本地 PaddleOCR 模型配置

若要使用本地离线 OCR，需将模型文件放入以下目录：

```
runtime/models/
├── text_detection/        ← 文字检测模型
├── text_recognition/      ← 文字识别模型
└── textline_orientation/  ← 文字方向模型（可选）
```

配置完成后，在 OCR 引擎下拉框选 **「本地 PaddleOCR」** 或 **「自动」** 即可使用。

## 七、常见问题

**Q：点击「自动生成候选帧」后没有反应？**
A：请先导入视频，确认左侧已显示视频信息后再操作。

**Q：提取结果为空？**
A：
- 视频画面中可能没有文字
- 所有候选帧均未勾选
- 网络问题导致 API 调用失败（查看右侧操作日志）

**Q：提示「OCR 引擎不可用」？**
A：检查 `runtime/api_config.json` 中 `enabled` 是否为 `true`，`api_key` 是否正确。

**Q：9:16 竖屏视频预览区显示异常？**
A：当前版本已支持 9:16 竖屏，预览区会自动按视频比例调整高度。若仍异常，请确认使用的是最新版本代码。

**Q：提取速度很慢？**
A：云端 API 每帧约需 3-8 秒。切换至本地 PaddleOCR 后速度会大幅提升（需提前配置模型）。

**Q：视频预览显示黑屏？**
A：部分 HEVC/H.265 编码视频预览可能异常，但不影响帧提取和 OCR 功能。

**Q：滚动模式为什么不可用？**
A：滚动模式将在 Build-3 版本实现，当前版本暂未开放。
