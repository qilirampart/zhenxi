# 帧析使用说明

## 软件定位

帧析用于从视频、图片和部分图文页面中提取可复制文本，适合处理短视频字幕、提示词、海报图、公众号图文等内容。

当前软件重点支持：

- 导入本地视频
- 导入本地图片
- 通过分享链接下载并导入素材
- 对单帧或多帧做 OCR 提取
- 查看结果并导出 TXT / JSON

## 推荐工作流

### 1. 导入素材

你可以使用以下入口：

- 导入视频：适合本地已有视频文件
- 导入图片：适合已经提前截好的图
- 链接下载：粘贴平台分享文本或链接，下载后自动导入

### 2. 选择提取方式

- 视频素材：先在提取页选帧，再执行 OCR
- 图片素材：导入后可直接提取
- 如果只想识别局部区域，可先框选区域，再提取

### 3. 查看结果

提取结果会进入结果页，可直接查看、复制、导出。

## 当前支持的平台能力

已集成或已具备可下载/解析能力的平台包括：

- 抖音视频
- 快手视频
- 小红书笔记 / 视频
- 哔哩哔哩视频
- 微信公众号文章内容

说明：

- 公众号当前以提取正文文本和图片为主，不以视频号完整下载为目标
- 不同平台会因分享口、地区、登录态、风控策略不同而出现偶发失败

## OCR 模式说明

### 推荐：云端 OCR / API

更适合当前版本，原因：

- 识别效果通常更稳定
- 不依赖本地模型环境
- 遇到本地推理兼容问题时更省事

### 兼容保留：本地 OCR

本地 OCR 目前仅作为兼容选项保留，不建议优先使用。

原因：

- 需要本地模型文件
- 启动更重
- 机器环境差异会影响成功率
- 某些环境下资源占用更高，但效果未必优于云端

建议：

- 默认优先使用云端 OCR
- 只有在离线环境或 API 不可用时，再尝试本地 OCR

## API 配置说明

软件支持最多 5 个 API 通道：

- 可手动指定首选通道
- 可测试连通性
- 可在首选失败时自动降级到后续可用通道

建议配置方式：

- 至少保留 2 个可用通道
- Base URL 必须包含接口前缀，例如 `/v1`
- 模型名必须与服务端实际可用模型一致

如果测试连接返回 200，但提取仍失败，常见原因有：

- 返回格式和 OpenAI 兼容格式不一致
- 模型本身不支持图片 OCR 场景
- API Key 可用，但模型权限不足

## 手动框选区域

适用场景：

- 只识别字幕区
- 排除水印、头像、背景装饰
- 降低无关文字干扰

使用建议：

- 尽量框得紧一些
- 避免一次框选过大区域
- 视频内容变化明显时，重新检查选区是否还对得上

## 提取结果与文件位置

软件的输出默认保存在项目目录下的 `output` 中。

常见目录：

- 下载素材：`output/downloads`
- 截图与提取中间文件：`output/screenshots`
- 导出结果：通常按你导出时选择的位置保存

如果是图文类内容，结果页中也会显示本地保存路径。

## 常见问题

### 1. 为什么本地 OCR 没有云端效果好？

这是正常情况。当前版本更偏向实用优先，云端 OCR 在多数场景下更稳，本地 OCR 只是保留备用口子。

### 2. 为什么平台链接偶尔下载失败？

常见原因包括：

- 分享内容不完整
- 平台临时风控
- 链接失效或被重定向
- 当前网络环境无法访问目标资源

### 3. 为什么测试 API 成功，但正式提取失败？

测试连接只验证连通性，不代表模型一定能正确完成 OCR 提取。

### 4. 为什么结果页没有内容？

请优先检查：

- 是否真的选中了帧
- 是否框到了正确区域
- 当前 OCR 通道是否可用
- 当前素材本身是否存在清晰可识别文字

## 当前建议

如果你的目标是稳定提取，建议优先采用这套顺序：

1. 先用链接下载或导入本地素材
2. 优先用云端 OCR
3. 必要时再用手动框选缩小识别区域
4. 本地 OCR 仅在 API 不可用时再尝试
